# ⚡ 超高速模式使用指南 (Ultra-Fast Mode)

## 📋 什么是超高速模式？

超高速模式 (`ultrafast`) 是一个**完全不需要调用 LLM**的快速论文检索方案，具有以下特点:

- ✅ **零 Token 成本** - 不使用任何付费 API
- ✅ **10-20 倍速度提升** - 几分钟内完成论文抓取
- ✅ **简单可靠** - 纯本地 BM25 算法，无外部依赖
- ✅ **适合日常监控** - 每天快速浏览最新论文

### 适用场景

| 场景 | 推荐模式 | 原因 |
|------|---------|------|
| 每天快速浏览最新论文 | ⭐ 超高速模式 | 速度快，成本低 |
| 每周深度分析一次 | 标准/快速模式 | 需要 LLM 摘要和评分 |
| 月初全面回顾 | 完整模式 | 覆盖更广的时间范围 |
| API 额度用完时 | ⭐ 超高速模式 | 零 API 依赖 |

## 🔧 高级用法

### 定时自动运行

如果您希望每天自动运行超高速模式，可以修改工作流配置文件:

编辑 `.github/workflows/daily-paper-reader-ultrafast.yml`,添加定时触发器:

```yaml
on:
  workflow_dispatch:  # 手动触发
    inputs:
      fetch_days:
        description: "回溯抓取天数（默认 3 天，极速模式）"
        required: false
        default: "3"
  
  schedule:           # 定时触发 (UTC 时间)
    - cron: '0 2 * * *'  # 北京时间每天早上 10 点运行
```

> 💡 **注意**: GitHub Actions 的 `schedule`使用的是 UTC 时间，需要根据时区转换

常见定时表达式:

```bash
# 每天早上 8 点 (UTC 时间 0 点)
- cron: '0 0 * * *'

# 工作日早上 9 点 (UTC 时间前一天 21 点)
- cron: '0 21 * * 1-5'

# 每周一上午 10 点 (UTC 时间周一凌晨 2 点)
- cron: '0 2 * * 1'
```

---

### 自定义安装依赖

如果您的检索逻辑需要额外的 Python包，可以在工作流中添加:

```yaml
- name: Install extra deps
  run: |
    pip install --quiet pandas numpy scikit-learn
```

添加到 `Install fast deps` 步骤之后即可。

---

### 并行运行多个工作流

您可以同时运行超高速模式和标准模式:

1. **超高速模式**: 每天运行，快速获取最新论文
2. **标准模式**: 每周运行一次，获取带 AI 摘要的深度分析

两者互不冲突，生成的内容会自动合并到 `docs` 目录中。

## ❓ 常见问题 FAQ

### Q1: 为什么我的工作流失败了？

**常见原因及解决方法:**

1. **Secrets未配置**
   ```
   Error: Missing required secrets
   ```
   
   **解决**: 确保已在 Settings → Secrets and variables → Actions 中配置必要的密钥
   
2. **Git 推送失败**
   ```
   Error: Failed to push to main
   ```
   
   **解决**: 检查工作流中的 git merge逻辑，或手动拉取最新代码后再试

3. **Python 依赖安装失败**
   ```
   Error: Could not find a version that satisfies the requirement xxx
   ```
   
   **解决**: 检查 `requirements.txt` 是否存在，或在工作流中显式安装包名

### Q3: 能否同时保留历史论文记录？

可以的！超高速模式只会向 `docs` 目录追加新内容，不会删除已有论文。

如果需要清理旧论文，可以使用后台管理面板的"重置内容"功能，或者手动删除 `docs` 目录下的子文件夹。

---

### Q4: BM25 分数是什么意思？

BM25(Best Matching 25)是一种经典的文本相似度算法，用于衡量论文与您设置的关键词之间的匹配程度。

- **分数越高** = 与您的需求越相关
- **分数越低** = 相关性较弱

一般参考标准:
- > 15分：高度相关⭐
- 10-15 分：中度相关
- < 10 分：弱相关

您可以在论文详情页看到每篇论文的 BM25 分数，帮助快速判断优先级。

---

### Q5: 如何让超高速模式也生成 AI 摘要？

目前超高速模式为了追求速度和零成本,**故意跳过了所有 LLM 调用**。

如果您偶尔需要 AI 摘要，建议:
1. 平时使用超高速模式快速追踪
2. 每月运行 1-2 次标准模式获取深度分析

这样既能控制成本，又能获得高质量的 AI 解读。

未来我们可能会推出"混合模式",在保证速度的前提下提供简化的 AI 摘要。敬请期待!

### 节省成本的技巧

1. **合理设置回溯天数**: 不要一次性抓取太久远的论文
2. **限制每日论文数量**: 避免无关论文占用注意力
3. **利用缓存机制**: 重复运行时会自动跳过已处理的论文
4. **错峰运行**: 避开 GitHub Actions高峰期可能更快

---

### 提高效率的技巧

1. **固定运行时间**: 养成每天同一时间查看的习惯
2. **善用颜色标记**: 用手机快捷键快速标注重要性
3. **导出到 Zotero**: 值得精读的论文及时保存
4. **定期清理**: 每季度归档一次旧论文，保持清爽
